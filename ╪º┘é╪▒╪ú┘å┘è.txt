دليل التشغيل خطوة بخطوة (اتبع الترتيب بالظبط)
====================================================

الجزء 1: إنشاء قاعدة البيانات (Supabase)
----------------------------------------
1. روح على supabase.com وسجّل دخول بحسابك (لو عندك حساب من قبل).
2. دوس "New Project".
3. اختار اسم للمشروع (أي اسم)، وكلمة سر لقاعدة البيانات (احفظها في مكان آمن)،
   واختار المنطقة الأقرب لك (مثلاً Frankfurt أو أقرب منطقة متاحة).
4. استنى دقيقة لحد ما المشروع يخلص إعداد.

الجزء 2: إيقاف تأكيد الإيميل (عشان التسجيل يبقى فوري بدون تعقيد)
----------------------------------------------------------------
1. من القائمة الجانبية: Authentication -> Providers -> Email.
2. لو لاقيت خيار "Confirm email" مفعّل، بطّله (Disable) واحفظ.
   (ده يخلي أي حد يسجل حساب جديد يدخل على طول من غير ما يفتح إيميله يأكد).

الجزء 3: إنشاء الجداول (الخطوة المهمة)
----------------------------------------
1. من القائمة الجانبية: SQL Editor -> New query.
2. افتح ملف "schema.sql" الموجود في نفس المجلد ده، انسخ كل محتواه.
3. الصقه في مربع الكتابة في Supabase ودوس "Run" (أو RUN في الأسفل).
4. المفروض يظهرلك "Success. No rows returned" - يعني تمام.

الجزء 4: الحصول على مفاتيح الربط
----------------------------------------
تم بالفعل! ملف ".env.example" فيه بيانات مشروعك جاهزة، مش محتاج تعمل حاجة في الجزء ده.

الجزء 5: رفع المشروع على GitHub
----------------------------------------
1. اعمل Repository جديد وفاضي على GitHub (New repository)، سمّيه مثلاً
   "factory-app".
2. من صفحة الـ Repository، دوس "uploading an existing file" وارفع كل
   الملفات والمجلدات الموجودة هنا (ما عدا مجلد node_modules لو موجود،
   ومجلد dist، وملف .env - دول من غير ما تترفع، الموقع هيولّدهم لوحده).

الجزء 6: النشر على Vercel
----------------------------------------
1. روح على vercel.com وسجّل دخول (اربطه بحساب GitHub بتاعك لو مش مربوط).
2. دوس "Add New..." -> "Project".
3. اختار الـ Repository اللي رفعته دلوقتي (factory-app).
4. قبل ما تدوس Deploy، هتلاقي قسم اسمه "Environment Variables".
   ضيف فيه القيمتين دول (لازم تتكتب بنفس الاسم بالظبط):
   - الاسم: VITE_SUPABASE_URL       القيمة: (اللي نسخته من Supabase)
   - الاسم: VITE_SUPABASE_ANON_KEY  القيمة: (اللي نسخته من Supabase)
5. دوس "Deploy" واستنى دقيقة.
6. هيديك رابط زي: factory-app-xxxx.vercel.app - ده الرابط اللي تبعته
   لأي حد يشتغل بيه.

الجزء 7: أول استخدام
----------------------------------------
1. افتح الرابط، دوس "حساب جديد"، اكتب اسمك وإيميلك وكلمة سر واختار صفتك
   (مدير / محاسب / موظف إنتاج)، ودوس "إنشاء الحساب".
2. أي شخص تاني هيفتح نفس الرابط، يعمل نفس الخطوة باسمه هو، ويختار صفته
   هو، ويبقى شغال معاك على نفس البيانات فورًا.

ملاحظات مهمة
----------------------------------------
- أول شخص يسجل ممكن يختار "مدير" لنفسه، وباقي الفريق يختاروا صفتهم
  المناسبة. مفيش رقابة تمنع حد من اختيار "مدير" لنفسه - النظام مبني
  على الثقة بين أعضاء الفريق، مش تشفير أمان بنكي.
- لو عايز تعدّل حاجة في الكود بعد كده وترفعها تاني، أي تعديل ترفعه على
  GitHub بيتحدث تلقائيًا على نفس الرابط في Vercel من غير ما تعمل حاجة
  زيادة.
